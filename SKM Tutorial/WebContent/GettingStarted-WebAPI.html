Getting started with Serial Key Manager

A quick introduction to the basics of Serial Key Manager platform

Web API

In the previous tutorial we saw that keys have many different fields that can be customized. Imagine now that you are able to retreive these values from any application. Clearly, this would allow you to make many customization to the client software based on the key. The way this information can be accessed is through the Web API.

List of requests

ActionDescription
ValidateThis type of request is usually sent from your client software to check whether a key is valid.
ActivateThis works almost as Validate, except that this request can register machine codes and change the key.
Optional Field This will retreive an integer value that was associated with the key. This value can be either retreived or decreased.
Generate a KeyThis will generate a new key with specified propoties, eg. set time, features.
List of ProductsThis will return a a list of your current products.
Get Product VariablesThis will return some variables that are needed to perform Activation and Validation.

Remark

The requests above have different authentication levels, that is, the amount of information that is required to perform a request.

  • The ones marked in green do not require authentication at all.
  • Those marked with yellow require a private key (see your private key). You need to explicitly enable external key generation to make it work.
  • Those marked with blue require the username and the password that you use to log in into your account.

Summary

In this article, we've looked at all the options that can be specified in a key. We made some references to the Web API, which we will discuss even further in the next article!
